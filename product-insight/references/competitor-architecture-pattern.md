# 竞品分层架构对比分析模式

> 适用于云厂商/SaaS/平台型产品的竞品深度分析。来源：阿里云 OPC 全栈分析实践。

## 适用场景

当竞品是一个**多层次、多产品线的平台型产品**（如云厂商、SaaS 平台）时，逐层拆解比全局对比更有效。

## 分析框架

### 1. 分层拆解

将竞品的完整产品线按技术栈/业务栈拆分为 **4-6 层**，通常为：

| 层级 | 典型内容 | 分析重点 |
|------|---------|---------|
| 底层基础设施 | 芯片/算力/网络 | 自研 vs 采购、成本优势 |
| 平台层 | 模型/IaaS/PaaS | 开放性、生态绑定 |
| 中间件/工具层 | AI平台/CLI/SDK | 开发者体验、CLI原生化 |
| 应用层 | Agent/SaaS/模板 | 零门槛程度、开箱即用 |
| 生态层 | IM/社交/开源 | 渠道壁垒、网络效应 |

### 2. P0/P1/P2 差异化评级

对每层的每个产品进行优先级标记：

- **P0** — 独有/显著领先（竞品无对标或明显落后）
- **P1** — 有特色/行业成熟（与竞品持平或有微弱优势）
- **P2** — 行业标配（三家都有，无明显差异）

### 3. 竞品对照表

对每一层，列出目标竞品的主要对标产品：

```markdown
| 维度 | 目标产品 | 竞品A | 竞品B |
|------|---------|-------|-------|
| OPC专属套餐 | ✅ 首创三档 | ❌ 无 | ❌ 无 |
| CLI/Skill原生化 | ✅ 开源独创 | ⚠️ 非Agent原生 | ⚠️ 非Agent原生 |
| 端云协同 | ⚠️ 一般 | ✅ 强 | ❌ 弱 |
```

使用彩色标记：`✅` 领先 / `⚠️` 持平 / `❌` 落后。

### 4. SWOT 分析（每层/全局）

分别从 S（优势）、W（劣势）、O（机会）、T（威胁）四个角度总结。

### 5. 战略建议分层

按 P0/P1/P2 优先级输出行动建议：

```markdown
**P0 立即行动** — 可快速拉开差距的差异化方向
**P1 短期重点** — 需要补齐但难度中等的方向
**P2 中期布局** — 重要但优先级较低的方向
```

## 输出格式

推荐 HTML 报告（卡片式布局），包含：

1. Hero Header — 分析对象 + 结论摘要
2. KPI 网格 — 关键数字速览
3. 每层卡片 — 层名 + 色条 + 产品网格 + 竞品对比
4. 总评表 — 所有层的 P0/P1/P2 汇总
5. 战略建议卡片（P0/P1/P2 分类）
6. 一句话总结

## 常见陷阱

- **❌ 只列产品不对比** — 每层的产品必须标注与竞品的关系（领先/持平/落后）
- **❌ 忽略竞品未切入的领域** — 当竞品无对标产品时，标注「无对标」本身是重要结论
- **❌ 不分优先级** — P0/P1/P2 是让报告可执行的关键，不可省略
- **❌ 战略建议太泛** — 每条建议必须有具体的执行方向，而非「加强产品力」
