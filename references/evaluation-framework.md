# 详细测评框架

## 评分规则

每个子维度采用 5 分制，最后按权重折算为百分制。

### 评分细则 (0-5)

| 分数 | 标准 |
|------|------|
| 5 | 行业标杆，可被作为最佳实践引用 |
| 4 | 良好，不超过 1 个可优化点 |
| 3 | 合格，存在 2-3 个问题但不阻碍核心流程 |
| 2 | 较差，有明显问题影响用户完成任务 |
| 1 | 严重，存在阻断性问题，导致用户无法完成核心操作 |
| 0 | 完全缺失该方面的设计 |

> **资料类特殊说明**：评分时以「用户能否独立、顺利地根据文档完成任务」为核心判定依据。例如：步骤缺失关键验证方法 → 2 分（阻碍完成）；文档无前置条件说明导致操作失败 → 1 分（阻断）。

## 各维度详细评分项

### 视觉层（权重 20%）

#### V1. 品牌一致性 (0-5)
- 色彩使用是否一致（主色偏差 ≤5%）
- 字体栈是否统一（不超过 2 种字体族）
- 按钮样式是否统一（圆角、阴影、大小在同一层级一致）
- 图标风格是否统一（线性/面性不混用）

#### V2. 色彩系统 (0-5)
- 是否有明确的主色、辅色、中性色体系
- 文字与背景对比度是否满足 WCAG AA（≥4.5:1 正文，≥3:1 大文字）
- CTA 颜色是否与环境形成足够对比（至少 6:1）
- 是否考虑色盲友好（不仅靠颜色传递信息）

#### V3. 排版层级 (0-5)
- 是否有清晰的 H1→H2→H3→正文层级
- 字号梯度是否合理（H1 约 2em, H2 约 1.5em, H3 约 1.17em, 正文 16px+）
- 行高是否舒适（正文 1.5-1.75，标题 1.2-1.3）
- 段间距是否区分内容块

#### V4. 视觉重心 (0-5)
- 首屏 3 秒测试：能否快速识别「这是什么产品」「对我有什么价值」「下一步做什么」
- CTA 是否在视觉层级中处于最高优先级
- 是否有不必要的视觉干扰（自动播放视频、过量动效）
- 移动端首屏信息是否完整

#### V5. 图标与插画 (0-5)
- 图标是否有语义辅助或有实际信息传递
- 风格是否统一
- 是否过度使用装饰性元素

### 交互层（权重 30%）

#### I1. 导航清晰度 (0-5)
- 主导航 ≤7 项
- 当前位置有明确指示
- 3 次点击内可达任意页面
- 移动端导航可用（汉堡菜单/底部导航）
- 返回/Home 入口始终可访问

#### I2. 操作效率 (费茨定律) (0-5)
- 主 CTA 尺寸 ≥44×36px（桌）或 44×44px（移）
- 关键操作间距合理（相关近、无关远）
- 表单提交按钮靠近最后一个输入框
- 移动端触控目标间距 ≥8mm

#### I3. 选择复杂度 (希克定律) (0-5)
- 定价方案 ≤4 个
- 下拉选项 ≤10 项
- 有明确的推荐选项标记
- 长列表支持搜索/筛选

#### I4. 表单体验 (0-5)
- 标签明确且靠近对应输入框
- 实时校验有清晰反馈
- Tab 键顺序符合预期
- 必填/选填有区分
- 支持自动填充（HTML autocomplete 属性）

#### I5. 反馈与状态 (0-5)
- 可点击元素有 hover/active/focus 态
- 加载状态有明确指示（骨架屏/进度条/spinner）
- 操作成功/失败有明确提示
- 长时间操作显示预估时间或进度

#### I6. 错误与空状态 (0-5)
- 错误提示包含「出了什么问题」+「为什么」+「怎么解决」
- 空状态有引导操作（而非空白或仅有文字说明）
- 404/500 页面有回到首页的路径
- 输入错误焦点自动定位到错误字段

### 文案层（权重 25%）

#### C1. 标题与价值主张 (0-5)
- 主标题 5 秒内传达核心价值
- 副标题补充说明差异化优势
- 面向 SMB 客户的语言，避免技术术语
- 主标题长度 ≤20 字

#### C2. 卖点表达 (0-5)
- 功能描述转化为利益描述
- 每个核心卖点有对应的场景或数据支撑
- 技术优势有合理的通俗解释
- 不使用空洞的形容词（"强大的""领先的""卓越的"→需要数据替代）

#### C3. CTA 有效性 (0-5)
- 按钮使用动词+利益（"免费开始使用"优于"提交"）
- CTA 周围消除了用户疑虑（安全提示、无需信用卡、"3 分钟完成"）
- 主次 CTA 文案有明确区分（不是两个都是「了解更多」）
- 不同阶段的 CTA 匹配用户决策阶段

#### C4. 语气与一致性 (0-5)
- 全文语气统一
- 语气匹配目标客群（SMB→友好、务实、不浮夸）
- 数字、日期、标点格式统一
- 面向中国大陆客户时语言符合本地习惯

#### C5. 信任与安全感 (0-5)
- 有明确的隐私/安全承诺
- 订阅/付费前有金额确认
- 有「无需信用卡」「随时取消」等消除门槛的文案
- 价格说明无隐藏费用或模糊表述

### 转化漏斗（权重 25%）

#### F1. 购买路径长度 (0-5)
- 首页→购买完成 ≤5 步
- 路径中无强制注册/关注/跳转
- 路径中无不相关推荐或交叉销售干扰
- 有进度指示器

#### F2. 定价清晰度 (0-5)
- 价格在首屏可见或 1 次滚动内可见
- 不同套餐的差异用对比表一目了然
- 计费方式有通俗解释（按量、包月、包年有什么区别）
- 有常见定价问题 FAQ

#### F3. 信任背书 (0-5)
- 有客户 Logo 或案例
- 有具体数据（服务客户数、uptime 等）
- 有行业认证/安全合规标识
- 背书元素位置合理（CTA 附近，而非底部）

#### F4. 决策成本 (0-5)
- 提供免费试用/演示/沙箱环境
- 有退款/退订保证
- 有对比工具帮助用户选择
- 支持不注册即可体验核心功能

#### F5. 紧迫感与行动力 (0-5)
- 有合理的紧迫感（限时优惠、限量活动）
- 紧迫感设计不过度（不虚假倒计时，不夸大稀缺性）
- 首次访问有新用户专属优惠

## 页面类型识别（测评第 0 步）

测评前必须根据 URL 和页面内容判断页面类型，启用对应的评估策略。

| 类型 | 识别特征 | 典型 URL | 核心目标 |
|------|---------|---------|---------|
| **品牌宣传类** | Hero大图、品牌故事、客户案例、价值观陈述 | 首页 `/`、`/about`、`/customers` | 品牌认知与情感共鸣 |
| **购买类** | 价格表、CTA按钮、套餐对比、试用入口 | `/pricing`、`/buy`、活动页 | 购买转化 |
| **操作类** | 表单密集、数据表格、操作按钮、侧边菜单 | 控制台、`/console`、`/dashboard` | 操作效率与任务完成 |
| **资料类** | 文档层次、代码块、步骤说明、左侧目录 | `/docs`、`/help`、`/guide` | 信息查找与自助解决 |

**识别流程：**
1. 看 URL：`support.*.com` / `/docs/` → 资料类；`/pricing` / `/buy` → 购买类；`/console` / `/dashboard` → 操作类
2. 看元素：价格表+CTA → 购买类；表单+数据表 → 操作类；文档目录 → 资料类；Hero大图+案例 → 品牌宣传类
3. 看意图：卖产品 → 购买类；讲故事 → 品牌宣传类；帮用户做事 → 操作类；帮用户学习 → 资料类

## 各页面类型的评估策略

### 类型一：品牌宣传类

**聚焦：品牌调性、视觉冲击力、故事感染力。不涉及购买转化。**

| 维度 | 权重 | 检查侧重 |
|------|------|---------|
| 🎨 视觉层 | **35%** | 品牌一致性、Hero冲击力、案例视觉化呈现 |
| 🖱 交互层 | **20%** | 浏览体验（滚动、动画、导航），非操作效率 |
| ✍️ 文案层 | **35%** | 品牌故事感染力、价值主张独特性、客户引语真实感 |
| 📈 转化漏斗 | **10%** | 仅轻度转化（订阅/关注/预约Demo） |

```
综合得分 = V_avg×20 × 0.35 + I_avg×20 × 0.20 + C_avg×20 × 0.35 + F_avg×20 × 0.10
```

**品牌宣传类专属清单：**
- [ ] 品牌色/Logo/字体是否统一且符合品牌调性？
- [ ] Hero 区域是否在 3 秒内传递品牌核心信息？
- [ ] 客户案例是否以真实人物+具体数据呈现？
- [ ] 文案是否有情感共鸣或独特观点（避免陈词滥调）？
- [ ] 页面是否有明确的品牌差异化（"为什么选我们"）？
- [ ] 是否有可分享的内容（洞察报告、行业观点）？
- [ ] 是否有轻度互动入口（订阅/关注/预约）且不破坏浏览体验？


### 类型二：购买类

**聚焦：文案说服力与购买转化率。C+F 权重合计 65%。**

| 维度 | 权重 | 检查侧重 |
|------|------|---------|
| 🎨 视觉层 | **15%** | 品牌一致即可，不求惊艳 |
| 🖱 交互层 | **20%** | CTA尺寸与位置（费茨）、表单、反馈 |
| ✍️ 文案层 | **35%** | 标题吸引力、卖点利益化、CTA有效性、信任消除 |
| 📈 转化漏斗 | **30%** | 购买路径、定价、信任背书、决策成本、紧迫感 |

```
综合得分 = V_avg×20 × 0.15 + I_avg×20 × 0.20 + C_avg×20 × 0.35 + F_avg×20 × 0.30
```

**购买类专属清单：**
- [ ] 价格是否首屏可见或 1 次滚动内可见？
- [ ] CTA 是否使用动词+利益（"免费试用"而非"提交"）？
- [ ] CTA 周围是否有风险消除文案（"无需信用卡""随时取消"）？
- [ ] 是否有信任背书（客户 Logo、案例数据、认证标识）？
- [ ] 购买路径是否 ≤5 步且有进度指示器？
- [ ] 是否有免费试用/演示入口降低决策门槛？
- [ ] 是否有合理的紧迫感设计（限时优惠、新用户福利）？
- [ ] 定价方案是否 ≤4 个且有明确推荐标记？
- [ ] 是否有定价 FAQ 解答常见疑虑？
- [ ] 文案是否以「你」而非「我们」为主语？


### 类型三：操作类

**聚焦：视觉逻辑清晰度与操作效率。V+I 权重合计 70%。**

| 维度 | 权重 | 检查侧重 |
|------|------|---------|
| 🎨 视觉层 | **30%** | 信息布局清晰度、数据展示易读性、操作按钮醒目度 |
| 🖱 交互层 | **40%** | 导航、费茨定律、表单、反馈、错误处理（全部加重） |
| ✍️ 文案层 | **20%** | 标签清晰度（按钮/菜单/表单）、错误提示有效性 |
| 📈 转化漏斗 | **10%** | 操作完成率（非购买场景） |

```
综合得分 = V_avg×20 × 0.30 + I_avg×20 × 0.40 + C_avg×20 × 0.20 + F_avg×20 × 0.10
```

**操作类专属清单：**
- [ ] 信息架构是否清晰（导航≤7项，3次点击可达）？
- [ ] 关键操作按钮是否 ≥44×44px（移）或 ≥32×32px（桌）？
- [ ] 表单是否有实时校验和明确的错误提示？
- [ ] 操作后是否有即时反馈（loading态、成功/失败提示）？
- [ ] 数据表格是否支持排序、筛选、搜索？
- [ ] 批量操作是否可用（勾选+批量删除/导出）？
- [ ] 空状态是否有引导操作（而非空白）？
- [ ] 是否有撤销/返回机制（用户控制与自由）？
- [ ] 文案标签是否简洁明确（≥95% 用户一次看懂）？
- [ ] 加载时间是否可接受（骨架屏/进度条缓解等待焦虑）？


### 类型四：资料类

**聚焦：内容质量——读得懂、跟着做、能验证。帮助用户基于文档独立完成任务，从「看懂」到「做到」再到「确认做对了」。**

资料类页面（技术文档、帮助中心、操作指南）不适用视觉/交互/转化漏斗的通用评测维度。文档的核心价值在于：用户能否独立、顺利、无歧义地完成文档所描述的操作。评测重心从「页面设计」转移到「内容工程」。

| 维度 | 权重 | 检查侧重 |
|------|------|---------|
| 📖 内容可读性 | **40%** | 标题精准、信息分层、语言可理解、代码/命令可执行 |
| 📝 内容精简度 | **25%** | 信息密度控制、冗余排查、开篇效率、章节聚焦 |
| 🔧 操作完整性 | **35%** | 步骤可执行、结果验证、修改与删除、异常处理、后续引导 |

```
综合得分 = R_avg×20 × 0.40 + C_avg×20 × 0.25 + O_avg×20 × 0.35
```

---

#### 维度一：📖 内容可读性（权重 40%）

用户能否轻松读懂文档，并理解每一步在做什么、为什么要这样做。

##### R1. 标题精准度 (0-5)
- 标题是否准确描述文档要解决的问题（如「使用 RDS 创建 MySQL 数据库」优于「数据库操作」）
- 是否避免使用「概述」「简介」「其他」等不传递信息的模糊标题
- 子标题是否反映对应章节的真实内容

##### R2. 信息分层 (0-5)
- 是否遵循「概览（本文做什么）→ 前置条件 → 操作步骤 → 结果验证 → 后续清理/下一步 → FAQ」的清晰层级
- 每个层级是否有明确的视觉区分（标题、编号、缩进）
- 读者是否能通过目录/标题快速跳转到所需章节

##### R3. 语言可理解性 (0-5)
- 是否使用主动语态、短句（单句 ≤40 字为佳）
- 技术术语首次出现时是否有解释或链接（如「VPC（虚拟私有网络，相当于你自己的局域网）」）
- 是否避免使用行业黑话或只有内部人员能理解的缩写
- 面向 SMB 用户时，是否用类比降低理解门槛（如「Bucket 就像一个文件夹」）

##### R4. 代码/命令可执行性 (0-5)
- 代码块是否有一键复制按钮
- 命令中的占位符是否可识别且有说明（如 `$REGION` 而非裸的 `cn-hangzhou`）
- 命令是否可在目标环境直接粘贴执行（无隐藏字符、无省略号）
- 多步命令是否按执行顺序排列，每段代码前有说明「这段代码做什么」

---

#### 维度二：📝 内容精简度（权重 25%）

信息是否精炼无冗余，用户能否以最短时间获取所需内容。

##### C1. 信息密度控制 (0-5)
- 每个章节是否控制在 3 屏以内；超过 3 屏的内容是否提供了折叠、分步或锚点导航
- 长步骤是否用分步向导（Step 1/2/3）拆解
- 是否有不必要的长篇背景介绍挤占了操作内容的空间

##### C2. 冗余排查 (0-5)
- 是否存在跨章节的重复段落（同一段说明在多个章节原样出现）
- 是否有无意义的开场白（如「随着云计算的发展……」类行业背景铺垫）
- 是否使用了过度修饰词（「非常强大的」「极其便捷的」「卓越的」→ 需要数据或删除）
- 配图是否有信息增量，还是纯装饰性截图

##### C3. 开篇效率 (0-5)
- 文档第一段（或摘要区）是否用一句话说明「本文档帮你解决什么问题」
- 前置条件/权限要求/资源限制是否在文档开头一次性列出
- 预估操作时长是否标注（如「完成本文约需 10 分钟」）

##### C4. 章节聚焦 (0-5)
- 每个章节是否有单一、明确的目的
- 是否存在跑题内容（如在安装指南中大段介绍产品优势）
- 「相关文档」链接是否与当前内容真正相关，而非无差别堆砌

---

#### 维度三：🔧 操作完整性（权重 35%）

用户能否按照文档成功执行操作、验证结果、处理异常、以及回滚/清理。

##### O1. 步骤可执行性 (0-5)
- 每一步是否使用明确的动宾短语（「点击 **创建实例**」而非「然后我们创建实例」）
- 每一步是否提供完整的输入/输出示例（输入什么参数、看到什么结果）
- 控制台操作是否包含明确的导航路径（「控制台 > 产品 > 实例列表 > 创建实例」）
- 界面元素引用是否精确（使用加粗按钮名而非模糊的「点击那个按钮」）

##### O2. 结果验证 (0-5)
- 每步关键操作后是否有明确的验证方法（如何确认操作成功）
- 验证方式是否具体可执行——命令行有预期输出示例，控制台有所见即所得的截图比对
- 多步流程是否有阶段性检查点（"至此，你应该能看到……如果没有，请检查第 2 步"）

##### O3. 修改与删除 (0-5)
- 是否覆盖了「如何修改已创建的配置」（而非只讲创建）
- 是否覆盖了「如何删除/释放资源」（防止用户创建后不知道如何清理而产生费用）
- 是否有回滚/撤消的说明（配置变更失败时如何恢复）
- 对于不可逆操作，是否有醒目的警告提示

##### O4. 异常处理 (0-5)
- 是否列出了本节操作最常见的 2-3 个错误及其原因+解法
- 错误信息是否被翻译为用户可理解的语言（「Error 403」→ 「账号没有权限执行此操作，请检查 RAM 授权」）
- 是否有排障指引或指向更详细的排障文档的链接

##### O5. 后续引导 (0-5)
- 文档完成后是否有明确的「下一步」引导（而非文末空白）
- 相关文档链接是否有选择、有说明（每条链接附一句话说明其用途）
- 是否提供反馈机制（「这篇文档对你有帮助吗？」）以及反馈后的跟进路径

---

**资料类专属清单：**
- [ ] 标题是否精准描述要解决的问题（非「概述」「简介」）？
- [ ] 首段是否用一句话说明「本文帮你解决什么」？
- [ ] 前置条件/权限/资源限制是否在开头一次性列出？
- [ ] 内容是否遵循「概览→前置条件→操作步骤→验证→清理→下一步」层级？
- [ ] 每步是否可独立执行（动宾+输入/输出示例）？
- [ ] 每步关键操作后是否有明确的验证方法（预期输出/截图）？
- [ ] 是否包含修改配置、删除资源、回滚操作？
- [ ] 是否列出常见错误 + 原因 + 解决方案？
- [ ] 技术术语首次出现时是否有解释或链接？
- [ ] 代码块是否可一键复制，占位符是否可识别？
- [ ] 是否有「下一步」引导和相关文档交叉链接？
- [ ] 整篇文档能否让一个非专家的用户独立完成任务？

> **评估方法论**：资料类评测的核心验证手段是「文档走查」——测评者实际跟随文档步骤操作一遍，记录所有卡住、困惑、出错的位置。仅仅「阅读文档」是不够的，必须「执行文档」。

## 改进建议分级标准

| 优先级 | 标准 | 举例 |
|--------|------|------|
| **P0 (阻断)** | 直接导致用户无法完成核心任务或产生合规/安全风险 | 购买按钮不可点击、支付报错无提示；文档缺少关键前置条件导致操作必定失败、命令示例有语法错误无法执行 |
| **P1 (严重影响)** | 显著影响转化率或用户满意度，违反关键 UX 原则 | CTA 尺寸 <32px、定价隐藏过深、无任何信任背书；文档步骤缺少结果验证、没有异常处理导致用户踩坑后无法自救 |
| **P2 (优化)** | 不影响核心流程，但优化后能明显提升体验 | 文案可以更吸引人、图标风格不够统一；文档标题可更精准、部分术语缺少首次解释、缺少修改/删除操作说明 |
