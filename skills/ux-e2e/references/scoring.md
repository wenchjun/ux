# 评分细则

> 执行步骤 2 评分时参考。基于 1-5 分制，逐维度给出具体评判标准。

---

## 综合评分标准

| 分数 | 等级 | 说明 | 行动建议 |
|------|------|------|----------|
| 5 | 行业标杆 | 体验优秀，可作为最佳实践参考 | 保持并持续微调 |
| 4 | 良好 | 整体流畅，有小幅优化空间 | 按 P2 优先级迭代 |
| 3 | 及格 | 基本可用，但存在明显问题 | 重点修复 P1 问题 |
| 2 | 较差 | 问题影响用户完成任务 | 需系统性改进 |
| 1 | 严重缺陷 | 存在阻断性体验问题 | 需立即修复（P0） |

---

## 一、视觉层评分（1-5分）

### 5分 标准
- 布局网格清晰，信息层级三秒可扫描
- 品牌色彩统一，文字对比度全部通过 WCAG AA（4.5:1）
- 字体层级分明（标题/正文/辅助文字至少有三级明显区分）
- 图片/图标风格统一且语义清晰
- 移动端/平板/PC 三端均有适配，无横向滚动

### 扣分项

| 扣分 | 问题 | 示例 |
|------|------|------|
| -0.5 | 部分区域信息密度过高，不易扫描 | 大段文字无分段/无列表 |
| -0.5 | 色彩对比度不足（浅灰文字） | 辅助文字 #ccc 在白色背景 |
| -0.5 | 移动端布局断裂（横向滚动） | 表格/图片溢出 |
| -1.0 | PC/移动端视觉不一致 | 按钮颜色在两端不同 |
| -1.0 | 图片无 alt 属性（a11y 问题） | 主要配图无替代文字 |
| -1.5 | 信息层级混乱，核心信息不突出 | 标题与正文视觉权重接近 |
| -2.0 | 页面渲染错误（元素重叠/错位） | JSON数据可视化、布局错乱 |

---

## 二、交互层评分（1-5分）

### 5分 标准
- 主 CTA 按钮尺寸 ≥ 44×44px（移动端）、间距 ≥ 8px（费茨定律）
- 选择数量合理（导航 ≤ 7、套餐 3-5 个、表单选项适度分组）（希克定律）
- 所有操作有即时反馈（加载态、成功态、错误态）
- 危险操作有确认弹窗（删除、退订）
- 支持关键操作撤销/返回
- 表单实时校验，错误信息明确（非 "Error 500"）

### 扣分项

| 扣分 | 问题 | 示例 |
|------|------|------|
| -0.5 | 按钮 hover/active 态缺失 | 按钮点击无视觉反馈 |
| -0.5 | 表单无实时校验 | 提交后才提示格式错误 |
| -1.0 | 关键操作需要过多步骤 | 购买需要 5+ 步 |
| -1.0 | 移动端按钮过小或过密 | 两个链接间距 < 4px |
| -1.5 | CTA 按钮需滚动才能发现 | 购买按钮在第三屏 |
| -1.5 | 导航层级过深 | 需要 4+ 次点击到达目标 |
| -2.0 | 致命操作无二次确认 | 删除按钮直接生效 |
| -2.0 | 页面主要功能 JS 依赖无降级 | JS 禁用后白屏 |

## 三、文案层评分（1-5分）

### 5分 标准
- 核心标题 5 秒内传达产品价值
- CTA 文案明确可预期（"开始免费试用" > "提交"）
- 所有专业术语有解释（tooltip 或链接）
- 价格展示完整（含税/不含税、原价/现价、单位清晰）
- 中文表达自然，无翻译腔

### 扣分项

| 扣分 | 问题 | 示例 |
|------|------|------|
| -0.5 | 同一概念跨页面术语不统一 | 详情页叫"套餐"，购买页叫"计划" |
| -0.5 | 技术术语未解释 | SMB 客户看不懂"VPC""CDN" |
| -1.0 | 按钮文案模糊 | "了解更多"过于频繁 |
| -1.0 | 错误提示不友好 | "500 Internal Server Error" |
| -1.5 | 价值主张不清晰 | 看了 10 秒不知道产品做什么 |
| -1.5 | 价格信息不透明 | 隐藏税费、不含必要费用 |
| -2.0 | 中文翻译腔严重 | 明显机翻痕迹 |

## 四、信息架构层评分（1-5分）

### 5分 标准
- 内容按用户心智模型分类（而非内部组织架构）
- 面包屑/侧边栏导航清晰，用户知道自己在哪
- 核心信息首屏可见（无需滚动）
- 搜索功能可用（如果内容多）
- 相关页面间有清晰的导航链接

### 扣分项

| 扣分 | 问题 | 示例 |
|------|------|------|
| -0.5 | 缺少面包屑导航 | 深层页面无位置指示 |
| -1.0 | 核心信息需要滚动多次 | 价格在第三屏 |
| -1.0 | 分类不符合用户预期 | "帮助"在"关于我们"下面 |
| -1.5 | 信息重复/矛盾 | 详情页和购买页的价格不一致 |
| -2.0 | 搜索功能缺失（内容多时） | 大量文档但无法搜索 |

---

## 五、战略层评分（1-5分）

### 5分 标准
- 页面清晰服务于用户核心任务（了解产品/购买/使用）
- 商业目标与用户需求平衡（如注册时机合理）
- SMB 适配度高：低门槛、透明、易上手
- 信任元素到位：案例、资质、退款政策

### 扣分项

| 扣分 | 问题 | 示例 |
|------|------|------|
| -0.5 | 过早强制注册 | 浏览套餐就需要登录 |
| -1.0 | 缺少客户案例/信任背书 | 新用户无参考 |
| -1.0 | 退款/退订政策不清晰 | SMB 用户担心锁定 |
| -1.5 | 入门门槛过高 | 最低套餐价格远超竞品 |
| -2.0 | 体验设计背离用户目标 | 用户想了解功能但一直被推向购买 |

---

## 六、SMB 适配度专项评分（1-5分）

| 扣分 | SMB 特别关注点 |
|------|----------------|
| -0.5 | 定价无明显"入门档"（≥ ¥50/月，中小企业可接受范围） |
| -0.5 | 没有免费试用或低价体验降低决策门槛 |
| -1.0 | 技术术语过多，非技术背景的决策者难以理解 |
| -1.0 | 缺少中小企业客户案例或应用场景说明 |
| -1.5 | 购买流程复杂，需联系销售（SMB客户倾向自助） |
| -2.0 | 产品定位完全偏向大企业，对SMB无差异化考量 |

## 问题严重程度定义

| 级别 | 标签 | 定义 | 处理要求 |
|------|------|------|----------|
| P0 | 🔴 阻断 | 导致用户无法完成核心任务（购买/使用） | 24小时内修复 |
| P1 | 🟡 重要 | 显著降低用户体验和转化率 | 1-2周内修复 |
| P2 | 🟢 优化 | 影响体验细节和满意度 | 纳入迭代计划 |
