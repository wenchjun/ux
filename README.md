# UX Evaluator — 网页用户体验测评

## 这是什么？

一个面向**公有云/SaaS 产品**的系统化网页体验测评工具。输入一个 URL，输出一份专业的 HTML 测评报告——可以直接作为产品优化需求文档使用。

## 解决什么问题？

产品团队和设计师经常需要对网页做体验评审，但传统方式有两个痛点：

- **凭感觉评价**："这个按钮太小了""文案不够好"——没有方法论支撑，难以说服 stakeholders
- **交付物不可用**：口头反馈或零散文档无法直接驱动开发排期

这个技能把 UX 评审变成一门"手艺"：**每个结论都有原则依据，每条建议都具体到改什么→改成什么→为什么**，输出一份可以发给产品团队的正式报告。

## 核心设计

### 视角

**SMB（中小企业）客户视角**。始终追问："一个非技术背景的中小企业主，能顺利使用这个页面吗？"

为什么是 SMB？因为大客户有 IT 团队和专属客服，体验问题可以被人工兜底；而 SMB 客户一旦遇到障碍，直接流失。SMB 体验 = 产品的下限。

### 方法论

测评不是主观感受，而是基于成熟理论的结构化审查：

| 理论基础 | 具体引用 |
|---------|---------|
| Jesse James Garrett 用户体验五要素 | 战略层→范围层→结构层→框架层→表现层 |
| Alan Cooper《About Face 4》 | 用户心智模型、姿态理论、消除而非减少 |
| 费茨定律 | CTA 尺寸与位置的科学标准（T = a + b×log₂(D/W+1)） |
| 希克定律 | 选项数量与决策时间的关系（RT = a + b×log₂(n)） |
| 尼尔森十大可用性原则 | 系统状态可见性、用户控制与自由、错误预防等 |
| Fogg 行为模型 | B=MAT：行为 = 动机 × 能力 × 触发器 |
| Cialdini 说服原则 | 社会认同、权威、稀缺性、互惠、承诺一致性 |

### 四维度框架

所有页面按**四个维度、21 个子维度**进行打分（每项 0-5 分，加权汇总为百分制）：

| 维度 | 子维度数 | 核心问题 |
|------|---------|---------|
| 🎨 视觉层 | 5 项 | 看起来专业吗？3 秒内能找到重点吗？ |
| 🖱 交互层 | 6 项 | 操作顺畅吗？选择太多导致决策瘫痪吗？ |
| ✍️ 文案层 | 5 项 | CTA 有驱动性吗？卖点是利益还是功能列表？ |
| 📈 转化漏斗 | 5 项 | 从进来到购买要几步？价格看得见吗？信任够吗？ |

### 页面类型自适应

不同页面有不同的任务，不能用一个标准套所有。测评前先识别页面类型，启用对应的权重体系：

| 类型 | 典型场景 | 视觉 | 交互 | 文案 | 漏斗 |
|------|---------|------|------|------|------|
| 品牌宣传类 | 首页、关于页 | 35% | 20% | 35% | 10% |
| 购买类 | 定价页、活动页 | 15% | 20% | 35% | 30% |
| 操作类 | 控制台、管理后台 | 30% | 40% | 20% | 10% |
| 资料类 | 文档、帮助中心 | 20% | 25% | 30% | 25% |

## 报告长什么样？

生成的 HTML 报告采用 **Editorial Authority（编辑权威）** 设计风格——深色墨蓝基底 + 琥珀色强调 + Playfair Display 标题字体 + SVG 环形仪表盘，外观类似顶级咨询公司的分析报告。

包含 13 个模块：

1. Top Bar — 报告标识
2. Masthead — 页面信息 + 测评元数据
3. 综合评分 — 环形仪表盘 + 四维度进度条 + 判语
4. 测评逻辑清单 — 21 个子维度的完整评分表
5. 视觉层分析 — 5 张详细卡片（观察→原则→SMB影响）
6. 交互层分析 — 6 张详细卡片
7. 文案层分析 — 5 张详细卡片
8. 转化漏斗分析 — 5 张详细卡片
9. 核心优点 — 带 UX 原则依据的亮点总结
10. 改进建议 — P0/P1/P2 分级，含具体方案和预期效果
11. 改进清单总览 — 一表汇总所有问题
12. SMB 转化对照清单 — 页面类型专属排查项
13. Footer — 方法论说明

支持左侧 sticky 目录导航（scroll-spy 自动高亮）、移动端抽屉式导航、响应式布局。

## 评分标尺

| 分数 | 含义 |
|------|------|
| 90-100 | 行业标杆，SMB 客户可以无障碍完成购买 |
| 80-89 | 体验优秀，少数细节可优化 |
| 70-79 | 尚可使用，但存在明显的转化障碍 |
| 60-69 | 体验较差，SMB 客户很可能中途放弃 |
| <60 | 存在严重的可用性问题，需立即整改 |

## 限制

- **依赖 WebFetch**：测评质量受限于能抓取到的页面内容。SPA（单页应用）和需要 JavaScript 渲染的页面可能无法完整分析，此时会尝试 curl 或 WebSearch 作为补充
- **无法验证交互细节**：hover 态、动画、加载状态等需要浏览器实测的维度，基于推断评分并标注
- **桌面端为主**：移动端体验仅做推断性评价
- **Visual 评分偏保守**：没有真实渲染的页面，视觉层评分主要依赖品牌体系的一致性和已知设计规范
